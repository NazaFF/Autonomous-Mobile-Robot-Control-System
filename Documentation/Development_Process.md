# Development Process

This project followed an iterative engineering development methodology.

---

# 1. Requirements Definition

The objective was to design an embedded mobile robotic platform capable of operating both autonomously and under wireless remote control.

The robot should:

- Navigate safely
- Avoid obstacles
- Track a light source
- Support manual operation
- Integrate multiple sensors
- Be implemented on a custom PCB

---

# 2. Electronics Prototyping

All electronic subsystems were initially validated on a breadboard.

Subsystems tested:

- DC motors
- Motor driver
- Ultrasonic sensor
- Infrared receiver
- Light sensors

# 3. Embedded Software Development

Each subsystem was individually validated before system integration.

Examples include:

- Motor control
- Distance measurement
- Infrared communication
- Light sensor acquisition

The software was progressively integrated into a single firmware.

---

# 4. PCB Design

After validating the prototype, a dedicated PCB was designed using Autodesk Eagle.

Design goals included:

- Compact layout
